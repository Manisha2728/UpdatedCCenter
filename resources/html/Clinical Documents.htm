Set the Clinical Viewer Clinical Documents Front End Settings

Applies to: Clinical Viewer

  • History

Time Range Filter Services to restart

  • IIS(Clinical Viewer)
  • IIS(CVA)

The textbox determines the time Range used to filter the patient's Clinical Documents list when using the Show option.
The dropdown menu defines the time Unit used to filter the patient's Clinical Documents list. For example, if the Unit is "month" and the Range is 10, the patient's Clinical Documents over the past 10 months are displayed.
This configuration affects the Clinical Viewer and EHR Agent.
Default: 1 Year.

Display Options Services to restart

  • IIS(Clinical Viewer)
  • IIS(CVA)

Determines whether the "Show Cancelled" checkbox is displayed in Clinical Viewer.
If False: Show Cancelled checkbox is not displayed, and cancelled items are not displayed in the Clinical Viewer.
If True: Show Cancelled checkbox is displayed. Its default behavior is based on the "Show Cancelled is selected" configuration.
Default: True


Determines whether the Show Cancelled checkbox (if displayed) is selected.
Default: True

Determines the text to show for all External Documents on Disclaimer after the fixed text
Default: 'Show External Documents' and Fixed text: 'You are about to access patient documents from external system(s) for the following reasons:'

Determines whether external documents are retrieved and shown with other documents - when the user selects to Show All.
True: The External Documents checkbox will be selected automatically, so external documents will be retrieved and displayed with other documents.
False: The External Documents checkbox will not be selected automatically, so external documents will be not retrieved and displayed.
Default: True

Determines whether an External Document disclaimer will be displayed in EHR Agent when the user wants to access External Documents.

True:
In the EHR Agent a disclaimer message will be displayed before displaying the list of External Documents to the user. The user is expected to approve the disclaimer. The disclaimer will concatenate a fixed text with a configurable text.
Fixed Disclaimer Text:"You are about to access patient documents from external systems for the following reason(s):"
Configured Disclaimer Text: This will be the same text configured for the label of the Show External Documents checkbox in the Clinical Viewer. (The label is configured in the Show External Documents Checkbox Label configuration.) The text will be the same for both applications.
This configured text will also be displayed above the list of External Documents in the EHR Agent.

False:
The EHR Agent will not display the disclaimer text and will not display the configured text above the External Documents list. However, this text, if configured for Clinical Viewer External Documents checkbox, will be displayed in the Clinical Viewer.
Default: False








Determines whether in the Clinical Documents clinical view the presentation of the Document Completion Status is added to both the Preview and the Report (with title Report Status).
Note: This configuration applies only to a document opened in the Clinical Document clinical view, but not to the same document opened in an Imaging clinical view.
Default: False

Determines whether to show a Clinical Document folder even if it is empty.
If True, the empty folder is displayed.
If False, the empty folder is not displayed.
Default: False

Defines the Font Type in the preview of Clinical Documents.
Default: Courier New

Determines whether to sort documents by designation or by domain code.
Default: True

Defines the width of the Documents pane (in percentage) in relation to the Preview pane. The value range is 33%-70%.
Note: The horizontal width will be set after the user opens the Clinical Documents page. There is no need to click on External Documents.
Default: 33%

Defines the height of the External Documents area (in percentage) within the entire area of the Documents pane (including Internal and External Documents areas). The remainder of the Documents pane contains the internal Documents area (meaning, 100% minus this value). The value range is 0%-100%.
Note: This configuration takes effect only after External Documents is selected.
Default: 35%

Defines the default grouping method for the External Documents domain in Clinical Viewer.
Note that this does not affect the "Internal Documents" grouping in this domain.
Default: By Date

Filtering Clinical Data Services to restart

  • IIS(Clinical Viewer)
  • dbMotion Business Access Services (VPO, VPO1, VPO2, VPO3)
  • IIS(CVA)

Defines the document types (using the <code system>|<code>^<code system>|<code> format) that are filtered out of Clinical Documents. This configuration is mainly used to exclude Collaborate-bound e-Referrals from the clinical view, but it can also be used to filter out Imaging acts.
This configuration applies to Clinical Viewer, EHR Agent and Patient View.
Default: empty

Business Rules Services to restart

  • dbMotion Business Access Services (VPO, VPO1, VPO2, VPO3)
  • IIS(CVA)

Defines the code (using the <code system>|<code>^<code system>|<code> format) used to determine the priority order for displaying the Medical Staff type in the Document label.
For example, if the priority is Author, Authenticator, then the Author of the document is displayed. If there is no Author, the Authenticator is displayed.
If both exist, the first is displayed.
Default: 2.16.840.1.113883.5.90|AUT^2.16.840.1.113883.5.90|AUTHEN

Grouping Services to restart

  • dbMotion Business Access Services (VPO, VPO1, VPO2, VPO3)
  • IIS(CVA)

Determines the priority order for grouping:
Baseline, Local: First, group by baseline subdomain; if local code is unmapped, group by local subdomain.
Local, Baseline: First, group by local subdomain; if local code is under the root domain, and is mapped to a baseline code, group by baseline subdomain.
If local code is unmapped and is under a root domain, display under "Others".
This configuration applies to Clinical Viewer, EHR Agent and Patient View .
Default: Baseline, Local
