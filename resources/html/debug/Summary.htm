Display Options Services to restart

  • IIS(Clinical Viewer)
  • IIS(CVA)

Determines whether the domain grid title bars display messages in the Summary clinical view.
Default: True

Determines whether the Diagnosis grid or Problems grid is displayed in the Summary clinical view.
Default: Display Problems grid

Business Rules Services to restart

  • dbMotion Business Access Services (VPO, VPO1, VPO2, VPO3)
  • IIS(CVA)

Defines which Encounter Types to filter (to display) in the Encounter Summary grid. By default all Types are displayed.
To display only one or multiple specific Encounter Types, the <code system>|<code>^<code system>|<code> format must be used.
Default: ALL

Determines whether to filter (display) Medications whose status is Undefined.
If True, the medication is displayed.
If False, the medication is not displayed.
Default: True

Defines the code used to filter the Summary list (using the <code system>|<code>^<code system>|<code> format).
Default: 2.16.840.1.113883.5.14|cancelled^2.16.840.1.113883.6.96|inactive^2.16.840.1.113883.6.96|resolved^2.16.840.1.113883.6.96|73425007^2.16.840.1.113883.6.96|185981001^2.16.840.1.113883.5.1068|pending^2.16.840.1.113883.5.14|held

Defines the number of the patients' latest Encounter records to display in the Encounters Summary.
Default: 4

Defines the number of the patients' latest Allergy records to display in the Allergies Summary.
Default: 4

Defines the number of the patients' latest Problem records to display in the Problems Summary.
Default: 5

Defines the number of the patients' latest Lab records to display in the Labs Summary.
Default: All

Defines the number of the patients' latest Medication records to display in the Medications Summary.
Default: All

The Textbox determines the time Range for which to display Lab Events.
The Dropdown menu defines the time Unit for which to display Lab Events. For example, if the Unit is "Month" and the Range is 10, the patients' Lab Events over the past 10 months are displayed. Unit "All" displays all data (no filtering).
Notice: The final behavior is determined by the specific security role History Depth Configuration
Default: 1 Month

The Textbox determines the time Range for which to display Encounters.
The Dropdown menu defines the time Unit for which to display Encounters. For example, if the Unit is "Month" and the Range is 10, the patients' Encounters over the past 10 months are displayed. Unit "All" displays all data (no filtering).
Notice: The final behavior is determined by the specific security role History Depth Configuration
Default: All

The Textbox determines the time Range for which to display Medications.
The Dropdown menu defines the time Unit for which to display Medications. For example, if the Unit is "Month" and the Range is 10, the patients' Medications over the past 10 months are displayed. Unit "All" displays all data (no filtering).
Notice: The final behavior is determined by the specific security role History Depth Configuration
Default: All
