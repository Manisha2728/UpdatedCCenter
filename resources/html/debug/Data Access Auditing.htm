Set the CDR Instance Data Access Auditing Settings

  • History
Use this configuration to audit database access.

Changing configuration affects disk space consumption.

Changing configuration affects security attack tracking.

Enable database access auditing

Suspicious activity is any access to the database by non-dbMotion service accounts.
Application authorized users are dbMotion application service accounts.
Default: Enable suspicious activity auditing

Auditing files location and maximum storage limit

Auditing files are located on the disk path specified in the system parameters -\$_NODE_SQL_SERVER_AUDIT_PATH$


SQL Server audit files are kept in the file system.
Specify the maximum storage limit to keep disk space use under control.
After the limit reached, SQL Server overwrites the current files.
Default calculated according to expected amount of activity.

Authorired Users

Users listed are not monitored for suspicious activites.
Use comma seperated list of users in a format domain\user name, domain\user name, ...

Changes take effect immediately.
