Set Carequality Integration Settings

Applies to: Patient View

Carequality Integration Services to restart

  • dbMotion XDS IHE Services

Determines whether to enable or disable Carequality Integration.
For detailed implementation instructions, see the IHE Integration Implementation Guide.
Default: False

Defines the dbMotion Home Community ID (OID) for outbound requests to Carequality.
This value can be obtained from Allscripts Community Configuration Manager (CCM).
The OID format should be: urn:oid:n.n.n.n.n.n. For example: urn:oid:2.16.840.1.113883.3.57.1.3.0.2.
Default: Empty

Defines the Certificate Thumbprint for HTTPS connections to Allscripts Brokering Responding Gateway.
The Certificate must be downloaded from the Allscripts Community Configuration Manager (CCM) and installed on the Application server.
Default: Empty

URL to Allscripts Brokering Responding Gateway for IHE Transaction ITI-55 Cross Gateway Patient Discovery (XCPD).
Default: https://brokeringrespondinggateway- /iti55/<CQ Participant OID>/outbound

URL to Allscripts Brokering Responding Gateway for IHE Transaction ITI-38 Cross Gateway Query.
Default: https://brokeringrespondinggateway- /iti38/<CQ Participant OID>/outbound

URL to Allscripts Brokering Responding Gateway for IHE Transaction ITI-39 Cross Gateway Retrieve.
Default: https://brokeringrespondinggateway- /iti39/<CQ Participant OID>/outbound

Shared Participant List Services to restart

  • dbMotion XDS IHE Services
Define a list of Carequality participants to query in an outbound flow.
When users access a patient's Carequality documents in Patient View, all the Carequality participants listed are queried for documents,regardless of the user or patient in context, and regardless of the EHR used.

  • dbMotion XDS IHE Services
Define settings per EHR. Administrators can define Carequality participants to query that depend on the EHR used.
