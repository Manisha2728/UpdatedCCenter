Initiate General Settings Services to restart

  • dbMotion Business Access Services (VPO, VPO1, VPO2, VPO3)
  • VIA

The Initiate input parameter to define the member type in the MPI instance.
A Member is a set of demographic information that represents one individual (for example, a Person).
The value configured here (for example, Person) must be aligned with the Member value in the project's MPI.
Default: PERSON

Defines the entity type in the MPI instance.
An Entity is an attribute used for managing identities. It can have a value representing the relationship between two or more records or two members in a healthcare organization.
For example, it can represent multiple patient records sharing one Enterprise ID.
Default: PATIENT

Determines whether the Demographics search returns the Real (true) or Virtual index (false) as the leading patient index:
- Virtual Index: An index that is generated from a source system (root and extension) in an additional identifier field (memIdent) of the patient's record in Initiate. A record can hold multiple identifier fields.
- Real Index: An index that is generated from a full Initiate record.
Default: False

Determines whether filter mode is enabled for the Search/Match operation.
If enabled only known sources are searched (for better performance).
*In some cases of Initiate federation filter mode needs to be disabled in order to avoid searching for unknown sources.
Default: True

Defines the sort order of the patient indexes that are returned from the MPI to VIA (and to the clinical applications).
for example, "-getRecMtime" ("-" indicates descending order, and "getRecMtime" refers to the patient index modification time).
Default: -getRecMtime

Initiate Filter Settings Services to restart

  • dbMotion Business Access Services (VPO, VPO1, VPO2, VPO3)
  • VIA
For each patient index status in Initiate choose one of the following filters:
- Exclude: This patient index status will be excluded from the request of VIA to Initiate.
- VIA Input: This patient index status will be included in the request of VIA to Initiate but will not be forwarded to VIA consumers (for example, Clinical Viewer).
- VIA Input and Output: This patient index status will be included in the request of VIA to Initiate and will also be forwarded to VIA consumers.

Defines the Active filter.
Active status refers to the most current patient index in Initiate.
Default: VIA Input and Output

Defines the Overlay filter.
Overlay status refers to a patient index in Initiate that was overridden by another patient index.
Default: Exclude

Defines the Merged filter.
Merged status refers to a patient index that was merged to another patient index in Initiate.
Default: VIA Input

Defines the Deleted state filter.
Deleted status refers to a patient index that is logically deleted in Initiate.
Default: Exclude

Defines the minimum score assigned by Initiate to the Patient Search results below which the search results will be filtered out and not returned. The Score is calculated internally by Initiate on each index returned in Patient Search. It represents the quality (reliability) of the response. The purpose of the configuration is to filter out indexes that have a low quality score.
Default: 5
