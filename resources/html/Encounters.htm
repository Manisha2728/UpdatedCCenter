Time Range Filter Services to restart

  • IIS(Clinical Viewer)
  • IIS(CVA)

The textbox determines the time Range used to filter the patient's Encounters list when using the Show option.
The dropdown menu defines the time Unit used to filter the patient's Encounters list. For example, if the Unit is "month" and the Range is 10, the patient's Encounters over the past 10 months are displayed.
This configuration affects the Clinical Viewer and EHR Agent.
Default: 1 Year.

Business Rules Services to restart

  • dbMotion Business Access Services (VPO, VPO1, VPO2, VPO3)
  • IIS(CVA)

Determines whether to Enable the Episode Filter
When the value is True, only Encounters with End date = Null with Inpatient Code =2.16.840.1.113883.5.4|IMP are treated as open encounters (Episode). Emergency Code =2.16.840.1.113883.5.4|EMER will cause encounters to be treated as episodes based on the Duration of emergency encounter parameter (below) and other Encounter codes will be treated as closed encounters (Events).
When the value is False, every encounter with End date = Null is treated as an open encounter (an Episode). Duration of emergency encounter parameter is ignored.
Default: False

This configuration point is only relevant if the Enable Episode Filter parameter (above) is True.
Defines the duration, in number of hours, of an Emergency Encounter when there is no Discharge Date (Discharge Date = NULL).
Default: 24
