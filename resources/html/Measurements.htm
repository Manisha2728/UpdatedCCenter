Defines the priority order to display concatenated Body Weight measurements, when there are different types of UoM (Units of Measurement).
If the value is empty, all measurement types are displayed in a comma-separated list.
If the priorities are defined, the system displays the first (concatenated) measurement type if it exists. If it does not exist, the second priority is displayed, and so on.
If the concatenation configuration above is on, then the results will be concatenated.
The list of priorities must be displayed in the following format. If not, the display of vital signs will fail.
<first priority - code system>,<code>;<code system>,<code>|<second priority - code system>,<code>;<code system>,<code>

Note: When ticking this checkbox, please fill out the relevant configuration of UOM priory, in order for it to take effect.
Default: 2.16.840.1.113883.6.8,kg;2.16.840.1.113883.6.8,g|2.16.840.1.113883.6.8,[lb_av];2.16.840.1.113883.6.8,[oz_av]

Inpatient Measurements Services to restart

  • dbMotion Business Access Services (VPO, VPO1, VPO2, VPO3)
  • IIS(CVA)

Determines whether Inpatient Measurements are displayed in the Vital Signs domain in CV and EHR Agent.
True: Inpatient Measurements will be displayed in Vital Signs.
False: Inpatient Measurements will be filtered out and not displayed in Vital Signs.
Inpatient Encounter baseline codes: 2.16.840.1.113883.5.4|IMP^2.16.840.1.113883.5.4|ACUTE^2.16.840.1.113883.5.4|NONAC

Note that all Inpatient Measurements will be displayed. In extreme cases, this can increase system load and reduce performance and product usability. Therefore, we recommend loading only the relevant measurements to the CDR.
Default: False




Determines whether Emergency Measurements are displayed in the Vital Signs domain in CV and EHR Agent.
True: Emergency Measurements will be displayed in Vital Signs.
False: Emergency Measurements will be filtered out and not displayed in Vital Signs.
Emergency Encounter baseline codes: 2.16.840.1.113883.5.4|EMER

Note that all Emergency Measurements will be displayed. In extreme cases, this can increase system load and reduce performance and product usability. Therefore, we recommend loading only the relevant measurements to the CDR.
Default: False
