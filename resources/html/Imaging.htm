Set the Clinical Viewer Imaging Front End Settings

Applies to: Clinical Viewer

  • History

Time Range Filter Services to restart

  • IIS(Clinical Viewer)
  • IIS(CVA)

The textbox determines the time Range used to filter the patient's Imaging list when using the Show option.
The dropdown menu defines the time Unit used to filter the patient's Imaging list. For example, if the Unit is "month" and the Range is 10, the patient's Imaging over the past 10 months are displayed.
This configuration affects the Clinical Viewer and EHR Agent.
Default: 1 Year.

Display Filters Services to restart

  • IIS(Clinical Viewer)
  • IIS(CVA)

Determines whether the Imaging Details are displayed when printing an Imaging Report.
Default: True

Determines whether to show Imaging folder even if it is empty.
If True, displays empty folder.
If False, does not display empty folder.
Default: True


Imaging PACS Services to restart

  • IIS(Clinical Viewer)
  • IIS(CVA)
This configuration is used to access the PACS (Picture Archiving and Communication Systems) to display the selected Imaging records of the specific patient. The deviceId, useCode, schemeCode represents the system identity. The link to the system can be composed of both static and dynamic parts. These parts can be a function of the metadata (for example, use code) of the imaging record. Patient View only supports https:// URL for viewing PACS images.

Grouping Services to restart

  • dbMotion Business Access Services (VPO, VPO1, VPO2, VPO3)
  • IIS(CVA)

Determines the priority order for grouping:
Baseline, Local: First, group by baseline subdomain; if local code is unmapped, group by local subdomain.
Local, Baseline: First, group by local subdomain; if local code is under the root domain, and is mapped to a baseline code, group by baseline subdomain.
If local code is unmapped and is under a root domain, display under "Others".
This configuration applies to Clinical Viewer, EHR Agent and Patient View .
Default: Baseline, Local
