Search of CDR clinical documents Services to restart

  • dbMotion Business Access Services (VPO, VPO1, VPO2, VPO3)
  • IIS(CVA)
  • dbMotion Communication BOT Service

To conduct a document search, the Elasticsearch engine must index the documents that are loaded in the CDR. In Patient View, this setting enables a document search field in the Documents category.
Note: Users must be assigned a security task to conduct a search.
Default: False.



Systems are configured to search content and metadata, or search metadata only. Systems configured for metadata search only do not index content into Elasticsearch, meaning content is not available for searching. This setting specifies which systems to search using metadata only.
Note: The setting affects the indexing process, but not the search process, and does not affect documents already indexed in Elasticsearch.
* None: None of the systems are defined to support metadata search only. All systems support both metadata and content search for supported document types.
* Custom: Systems are defined by the organization to support metadata search only, using the document ID_Root of the clinical document. Use the pipe symbol [|] as a delimiter between systems.
* All: All systems are defined to support metadata search only.
Default: None.

Define source systems to metadata search only. Use the pipe symbol [|] as a delimiter between systems. Use the document Id_Root from clinicalDocument table.
Note: Changes to this setting does not affect past indexing.
Default: Empty

Define source systems to omit during indexing. Use the pipe symbol [|] as a delimiter between systems. Use the document Id_Root from clinicalDocument table.
Note: Changes to this setting does not affect past indexing.
Default: Empty

Search of External clinical documents Services to restart

  • dbMotion Business Access Services (VPO, VPO1, VPO2, VPO3)
  • IIS(CVA)
  • dbMotion Communication BOT Service

To conduct a document search, the Elasticsearch engine must index the documents that are loaded in the External documents repositories. In Patient View, this setting enables a document search field in the External Documents category.
Note 1: The search for external documents is supported only on metadata fields.
Note 2: Users must be assigned a security task to conduct a search.
Default: False.
